# Read Paper Skill

用于深度精读学术论文（PDF/LaTeX），重点回答两类问题：
- 为什么会有这种方法（动机与问题严重性是否成立）
- 这种方法怎么就能解决问题（方法与实验论证链条是否闭环）

## 适用场景

- 需要系统理解论文的研究动机、方法逻辑和贡献边界。
- 需要从审稿人视角评估论文的说服力与薄弱点。
- 需要产出可复用的结构化精读笔记。

## 输入约定

- 支持 PDF 或 LaTeX 源文件（单文件或多文件）。
- 用户可提供文件路径；未提供时，在当前工作目录定位论文主文件。

示例触发：

```text
请帮我精读 papers/example-paper.pdf
请深度精读 thesis/main.tex
```

## 输出结构

输出应包含以下部分，按顺序组织：

1. 题目与关键词解析
- 原文标题（完整提取）
- 关键词拆解（每个关键词对应的问题域/方法域/目标域）
- 一句话概述

2. 摘要与关键词提炼
- 原文摘要（完整提取，不删减）
- 关键词（原样提取）
- TL;DR（问题、方法、为何有效、有无该方法的差异）

3. 目录与章节组织分析
- 完整目录层级（章/节/小节）
- 各章节的功能定位与逻辑关系（承接关系要明确）

4. 背景与动机论证
- 背景部分服务于哪条核心论证
- 现有方法的具体问题与量化后果
- 从“指标下降”到“问题严重”的推导是否充分

5. 方法部分逐项解析
- 框架图/模块图：输入输出、关键机制、为何有效
- 伪代码/算法：核心逻辑、关键步骤、复杂度（如可得）
- 若涉及代码变换：说明关键转换通路

6. 实验设计与结果解读
- 每个图表单独说明“在验证什么问题”
- 平台、基线、指标、场景设置是否合理
- 结果中的定量提升、趋势、消融贡献

7. 相关工作与贡献边界
- 相关工作分类
- 与各类工作的联系与差异
- 论文贡献边界界定是否清晰

8. 讨论与审稿人视角总结
- Strengths（依据充分）
- Weaknesses（具体且可验证）
- Critical Questions（主张是否被充分验证）

## 关键检查规则

- 深度优先：必须解释“为什么/如何”，避免只复述“做了什么”。
- 完整性优先：不遗漏章节与关键图表。
- 准确性优先：标题、摘要、关键词需准确提取。
- 客观性优先：不盲目夸赞，不给分，不给 Accept/Reject 结论。

## 必须满足的约束

1. 摘要完整提取
- 摘要必须完整保留，不可截断或省略条目。

2. 目录完整提取
- LaTeX：从所有相关 `.tex` 中提取 `\\chapter`/`\\section`/`\\subsection`/`\\subsubsection`。
- PDF：从目录页或文档结构提取完整层级。

3. 图表逐项分析
- 不接受“实验结果表明有效”这类笼统描述。
- 每个图表都要说明：验证问题、结果、结论。

4. 审稿人视角问题清单
- 聚焦关键论证漏洞、实验不足、可复现性风险。
- 仅给问题与证据，不做总体评分。

## 最终交付格式

- 一份结构化精读报告。
- 结尾附“问题汇总”：
  - 主要问题类型与数量
  - 最优先修正项
  - 可执行的改进方向
