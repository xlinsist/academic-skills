# Polish Skill - 中文学术论文润色

此 skill 用于润色学术论文，自动识别并处理 LaTeX、PDF、Word 等常见格式，确保文本符合学术规范和写作标准。

## 使用场景

当你需要：
- 润色论文章节内容
- 检查拼写错误和基础语法问题
- 规范单复数、冠词使用
- 检查逻辑连贯性和严密性
- 规范缩写使用
- 统一术语风格和标题格式
- 统一标题大小写与连字符规则
- 提升学术表达质量
- 符合中文博士论文写作规范

## 润色流程

执行此 skill 时，按以下步骤进行（优先在原文上增量修改，不整段重写）：

### 1. **拼写与基础检查**
   - 扫描全文并修正明显拼写错误
   - 统一基础标点与空格问题

### 2. **缩写规范检查**
   - 统一缩写写法
   - 保持缩写前后一致
   - 缩写首次出现时给出全称（如 `LSTM (Long Short-Term Memory)`）

### 3. **语法与措辞润色**
   - 修正语法错误（含单复数、冠词、时态、搭配）
   - 删除冗余表达，压缩长句，提升信息密度
   - **重要**：润色时必须保持技术含义不变，只改进表达方式

### 4. **术语与风格统一**
   - 领域通用术语可直接使用；自创术语必须显式定义
   - 保持术语一致（如"RVV 架构"、"算子编译器"）
   - 同一概念在全文使用同一术语（如统一为"特征提取"，避免与"特征抽取"混用）
   - 统一文风并保持学术表达克制、严谨

### 5. **标题规范化**
   - 检查章节标题的大小写规范
   - 统一连字符使用规则
   - 确保标题格式一致性

### 6. **学术规范性**
   - 确保符合中文博士论文的严谨性要求
   - 弱化过于绝对或过度自信的表述

## 项目写作规范

本 skill 遵循以下写作规范：

- **人称使用**：避免第一人称"我们"，弱化第一人称表述
- **信息密度**：删减冗余，简化长句，提升信息密度
- **段落连贯**：避免分点，保持段落连贯，多用关联词和代词
- **标点规范**：不使用短横线从句、冒号/分号分点
- **举例方式**：不使用括号，将例子融入句子
  - ✅ 正确："在如 x86 的 AVX512 的定长向量架构中"
  - ❌ 错误："传统的定长向量架构（如 x86 的 AVX512）"
- **术语一致性**：保持术语统一（如"RVV 架构"、"算子编译器"、"分块编程模型"）
- **术语规范性检查清单**：
  - [ ] 领域通用术语直接使用，自创术语需显式定义
  - [ ] 缩写首次出现时给出全称（如 `LSTM (Long Short-Term Memory)`）
  - [ ] 同一概念在全文保持术语一致（如统一使用"特征提取"或"特征抽取"）
- **纯文本一致性**：像 vector.fma 帮我改成 \texttt{vector.fma}；对于原先用顿号“`”引用的，也用这个修改。
- **英文格式**：英文单词在中文原文中时左右两边各有一个空格
- **格式规范**：纯文本输出，不加粗、不斜体

## 输出内容

完成润色后，你将收到：

1. **详细分析报告**：列出发现的所有问题和修改建议
2. **修改后的文件**：自动应用润色建议到原文件
3. **改动总结**：概述主要修改点和改进方向

## ⚠️ 重要提醒

**不要打分！** AI打分比较主观，用户不会看评分。

只需要在报告结尾做一个**润色汇总**即可，列出：
- 主要修改类型和数量（拼写、语法、术语等）
- 改进最多的方面
- 整体润色效果概述

## 使用方法

```bash
/polish --file 路径/到/文件
```

或直接在对话中：

```
请帮我润色 研究内容一/概述.tex
请帮我润色 论文终稿.pdf
请帮我润色 第三章.docx
```
