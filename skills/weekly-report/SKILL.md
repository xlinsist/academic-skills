# Weekly Report

## 用途

- 用于周报、实习汇报、阶段性小结。
- 默认写成简洁中文，不展开成长篇技术报告。
- 用户没有指定格式时，直接使用四段结构：
  - `既定目标：`
  - `已完成：`
  - `进行中：`
  - `待讨论：`

## Workflow

1. 收集材料。
- 优先读取用户给的周报、日志、记录或聊天上下文。
- 只补充读取必要材料，不做过度展开。

2. 提炼四类信息。
- `既定目标`：这一阶段原本要达成什么。
- `已完成`：已经做完并可以确认的内容。
- `进行中`：正在推进但尚未完成的事项。
- `待讨论`：需要确认、协调、拍板的问题。

3. 输出成固定结构。
- 默认直接输出这四行标题，不额外加多余章节。
- 每一部分优先用 1-3 条短句表达，能短就短。

## 写作要求

- 用中文。
- 用简洁事实句，不写空话。
- 以“能直接拿去汇报”为准，不强制补技术细节。
- 没有信息的部分可写 `暂无`，不要编造。
- 只有在用户明确要求时，才补充函数、文件、命令、验证过程等技术展开。

## 默认模板

````markdown
既定目标：
- ...

已完成：
- ...

进行中：
- ...

待讨论：
- ...
````
